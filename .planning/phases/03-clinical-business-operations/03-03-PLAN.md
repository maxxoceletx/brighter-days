**Purpose:** Document the complete process for onboarding Nexus Billing (or any future third-party biller) to the practice, including Tebra access setup, data handoff, reporting expectations, and escalation procedures.

**Scope:** Triggered when onboarding a new biller OR when reviewing existing biller relationship. Nexus Billing is the current biller (already selected per locked decision).

**Roles and Responsibilities:**
| Role | Responsibility |
|------|---------------|
| Maxi (CTO) | Creates Tebra user, delivers onboarding packet, monitors biller performance, first escalation point |
| Valentina (Provider) | Approves diagnosis + CPT codes, reviews denied claims if escalated, final authority on clinical coding |
| Nexus Billing | Submits claims via Tebra, manages AR follow-up, reports denials same-day, provides monthly summary |

**Procedure -- Onboarding Steps (numbered checkboxes):**

1. Create Tebra user for biller:
   - [ ] Login to Tebra as System Admin (Maxi)
   - [ ] Navigate to Settings > Users > Add User
   - [ ] Enter: Biller contact name, work email
   - [ ] Role: Select "Biller" (NOT Billing Manager, NOT System Admin -- per HIPAA minimum necessary and locked decision)
   - [ ] Practice access: Enable for Valentina Park MD, PC only
   - [ ] Send invitation via Tebra (user sets own password)
   - [ ] Verify: Biller confirms access to charge entries and claims status

2. Prepare and deliver onboarding data packet:
   - [ ] NPI: 1023579513 (from credential inventory)
   - [ ] DEA: FP3833933 (from credential inventory)
   - [ ] Tax ID / EIN: [Valentina to provide]
   - [ ] All 17 payer IDs and contracts (export from payer-tracker-seed.sql -- Phase 2)
   - [ ] Standard psychiatry CPT codes: 90791, 99213, 99214, 99215, 90833, 90836, 90838
   - [ ] Fee schedule: [Valentina to provide or use Medicare rates as baseline]
   - [ ] Delivery method: secure channel (NOT email -- encrypted sharing or 1Password)

3. Document biller access in 1Password:
   - [ ] Record: user creation date, biller contact name, Tebra username
   - [ ] Category: Billing & Revenue > Nexus Billing (per 1Password vault spec -- Phase 2)

4. Confirm BAA status:
   - [ ] BAA with Nexus Billing: already signed (reference only -- do NOT create new checklist per locked decision)
   - [ ] Document BAA execution date and location in 1Password entry

5. Verify biller is operational:
   - [ ] Nexus confirms they can view charge entries
   - [ ] Nexus confirms they can view claims status
   - [ ] Submit test claim (if possible) or verify first real claim within 5 business days

**HIPAA Minimum Necessary Rationale (include as subsection):**
The Biller role exposes financial and administrative PHI (name, insurance, claim status) but NOT clinical PHI (diagnoses in clinical notes, prescriptions). This satisfies HIPAA's minimum necessary standard. If Nexus requests elevated access, Maxi reviews the specific need at the task level -- never upgrade the role.

**Biller role permissions (from research):**
- CAN: Manage charge entries, view claims status, view patient collections, view payout report, manage patient demographics (limited), manage appointments (limited), manage insurance, messages
- CANNOT: Access clinical notes, prescriptions, clinical documentation, user management, practice settings

**Reporting Expectations (per locked decision -- per-claim visibility):**
- Per-claim visibility: every claim status visible in Tebra (Nexus submits through Tebra -- all data flows back to practice)
- Denial notification: same-day contact when denial received (not batched monthly)
- Monthly summary report: submitted claims, paid claims, outstanding AR, denial rate
- Payment model: percentage of collections (already agreed)

**Escalation Path:**
1. Nexus Billing primary contact: [name + phone -- to be inserted at onboarding]
2. If unresolved after 5 business days: Valentina reviews claim directly in Tebra + contacts payer
3. If pattern of denials emerges: quarterly biller performance review meeting

**Decision Points:**
- Denial received: Nexus contacts Maxi same day -> Maxi evaluates if clinical review needed -> if yes, routes to Valentina -> Valentina reviews and provides documentation
- Biller requests elevated Tebra access: Maxi evaluates specific need -> accommodate at task level, never upgrade role -> document exception in 1Password
- Biller performance concern: review denial rate vs. industry benchmark (~5-10% for psychiatry) -> if consistently above 15%, schedule performance review -> if unresolved, evaluate biller replacement
