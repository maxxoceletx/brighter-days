**Section 8: Tebra Data Integration (DASH-05) and Billing Oversight Panel (DASH-06)**

**8A: Tebra Integration Architecture (DASH-05)**
- Purpose: define how appointment and billing data flows from Tebra into the dashboard
- Three-tier fallback strategy (from CONTEXT.md locked decision — spec each tier independently):

  **Tier 1 — Tebra SOAP API (Preferred):**
  - n8n Schedule Trigger fires every 15-30 min (configurable)
  - n8n HTTP Request node constructs SOAP envelope with RequestHeader (UserName, Password, CustomerKey from n8n encrypted credential vault)
  - Endpoints: GetAppointments (next 24 hours), GetTransactions (last 30 days for billing)
  - n8n Code node transforms SOAP XML response → JSON
  - PHI stripping (CRITICAL): n8n must extract ONLY: appointment_id, patient_first_name (strip last name), appointment_time, appointment_type. Never write last name, DOB, insurance info, or clinical data to Supabase.
  - n8n writes to Supabase tebra_appointments (upsert by appointment_id) and tebra_billing_summary (upsert by period)
  - For billing: n8n aggregates transactions into: claims_submitted (count), claims_denied (count), ar_current (sum where age < 30), ar_30 (sum where 30 <= age < 60), ar_60 (sum where 60 <= age < 90), ar_90plus (sum where age >= 90)
  - HIPAA: document both options from RESEARCH.md — (A) n8n as stateless passthrough with EXECUTIONS_DATA_SAVE_ON_SUCCESS=false, or (B) Keragon as HIPAA-native alternative. Developer chooses based on risk tolerance.
  - Include setup steps: get Customer Key from Tebra portal (Help > Get Customer Key), store in n8n credential vault, test with a single GetAppointments call

  **Tier 2 — Tebra CSV Export (Fallback):**
  - Tebra built-in reports exported as CSV (appointment report daily, billing report weekly)
  - n8n watches a shared folder (Google Drive or Dropbox) via File Trigger or Schedule trigger
  - n8n reads CSV, strips PHI (same rules as Tier 1), writes to Supabase
  - Downstream from Supabase is identical to Tier 1 — TD reads same tables
  - Manual step: Valentina or Maxi exports reports from Tebra on schedule. Spec should include the exact Tebra navigation path for export.

  **Tier 3 — Manual Data Entry (Last Resort):**
  - Dashboard panels 5 and 6 switch to editable mode
  - Panel 5: manual appointment entry form — Button COMPs + Text COMPs as input fields for first name + time
  - Panel 6: manual billing numbers form — Maxi enters weekly totals (claims submitted, denials, AR buckets)
  - Data written to Supabase via Web Client DAT POST (or via n8n webhook if preferred)
  - Flag: tebra_appointments.synced_at and tebra_billing_summary.synced_at show "Manual entry" instead of timestamp when data is manually entered

  **Tier selection display:** a small indicator in Panel 5 and 6 headers showing which tier is active: `[API]` / `[CSV]` / `[MANUAL]` — driven by dashboard_settings.tebra_integration_tier value

**8B: Panel 5 — Appointments Display (DASH-05 visualization)**
- Container COMP: panel_appointments (bottom-center position in grid)
- Data source: Supabase tebra_appointments WHERE appointment_date = TODAY (or next business day if weekend/holiday)
- Display: chronological list of "FirstName at HH:MM AM/PM" — example: "Alex at 2:00 PM, Jordan at 3:30 PM"
- PHI scope reminder: first name + appointment time only. No last names, no diagnoses, no insurance info on dashboard.
- Empty state: "No appointments scheduled" in dim CYAN
- Refresh: inherits global polling cycle
- Tier 3 mode: when tebra_integration_tier='manual', show editable form instead of read-only list

**8C: Panel 6 — Billing Oversight (DASH-06)**
- Container COMP: panel_billing (bottom-right of main area, left of actions row)
- Purpose: monitor third-party biller activity — oversight, not direct billing
- Data source: Supabase tebra_billing_summary for current and previous month
- Display layout (all aggregates, no patient-level detail):
  - Claims Submitted: count this month (with delta from last month: +N or -N)
  - Claims Denied: count this month + denial rate percentage
  - AR Aging buckets displayed as ASCII bar chart:
    ```
    Current  [########........] $X,XXX
    30-60d   [####............] $X,XXX
    60-90d   [##..............] $X,XXX
    90+d     [#...............] $X,XXX (RED if > $0)
    ```
  - Total AR: sum of all buckets
- Color coding: denial rate > 5% turns denial count AMBER; > 10% turns RED. AR 90+ > $0 always RED.
- Refresh: inherits global polling cycle
- Tier 3 mode: when tebra_integration_tier='manual', show editable form with fields for each metric
- HIPAA note: no patient names, no claim-level detail, no CPT codes on dashboard. Aggregates only.
