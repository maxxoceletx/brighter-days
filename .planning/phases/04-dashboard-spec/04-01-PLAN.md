- Panel borders: ASCII box-drawing characters (single line: `+-|`, double line for active/focused panels: `#=|`)

**Section 4: Panel 1 — Today / Morning Briefing (DASH-01)**
- Purpose: situational awareness view — what Valentina sees when she opens the dashboard each morning
- Container COMP: panel_today (top-left position in grid)
- Data sources (all from Supabase):
  - tebra_appointments WHERE appointment_date = TODAY — display as "First name at HH:MM" (PHI scope: first name + time only)
  - credential_alert_queue WHERE alert_level IN ('ALERT_30', 'ALERT_7', 'EXPIRED') — urgent items count
  - obligations WHERE status != 'completed' AND due_date <= TODAY + 7 — upcoming obligations
  - tebra_billing_summary WHERE period = current_month — billing snapshot (claims submitted, denials, AR total)
- Layout within panel: 4 stacked rows (appointments list, overdue count badge, upcoming obligations count, billing one-liner)
- ASCII art header: animated "BRIGHTER DAYS" banner using Text TOP with slow character-reveal animation on startup
- Animation states per Section 3 rules
- Refresh: inherits global 15-30 min polling cycle
- Empty state: "No appointments today" / "All clear — no overdue items" in dim CYAN text

**Section 5: Panel 2 — Compliance Status (DASH-02)**
- Purpose: at-a-glance red/yellow/green status for every tracked credential, license, BAA, and CAQH attestation
- Container COMP: panel_compliance (top-center position in grid)
- Data source: Supabase credential_alert_queue view (Phase 2 schema) — provides credential_name, alert_level, days_until_expiry, display_color for every credential with alert thresholds enabled
- Layout: vertical list of credentials, each showing:
  - [STATUS_INDICATOR] Credential Name — X days (or EXPIRED)
  - STATUS_INDICATOR is a colored ASCII block character: `[###]` in GREEN/YELLOW/RED
  - Color mapping (from Phase 2 alert-architecture.md): GREEN = 90+ days, YELLOW = 30-90 days, RED = 7-30 days, PULSING RED = <7 days or EXPIRED
- CAQH special handling: if expiry_date IS NULL, display as `[???] CAQH Attestation — VERIFY DATE` in amber with note "Last attestation date unknown — verify in CAQH portal" (per Pitfall 2 from RESEARCH.md)

- Scrollable if list exceeds panel height — use Scroll COMP or Python-driven vertical offset
- Animation: individual credential rows pulse when transitioning to a worse alert level (fade in-out on the row's background color 3 times, then hold)
- Click interaction: clicking a credential row opens a detail overlay showing credential_number, renewal_portal_url, vault_entry_ref, and last renewal date
- HIPAA physical security note: external monitor displaying this dashboard must be in a location where non-authorized individuals cannot view it
